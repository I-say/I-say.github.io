Regla de oro: Todo el contenido editable está en el bloque const ISAY = { al inicio del archivo index.html. Nunca necesitas tocar el resto del archivo para actualizar texto, eventos o proyectos.
1 Abrir el editor en GitHub
1
Ve a tu repositorio
Abre github.com/I-say/I-say.github.io en tu navegador. Puedes hacerlo desde el celular o la computadora.
2
Haz clic en index.html
Lo ves en la lista principal de archivos del repositorio.
3
Clic en el ícono de lápiz ✏️
Está en la esquina superior derecha del archivo. Esto abre el editor directo en el navegador.
4
Busca la sección que quieres editar
Usa Ctrl+F (o Cmd+F en Mac) para buscar el texto que quieres cambiar. El bloque editable empieza con const ISAY = { y termina con };
5
Guarda los cambios
Cuando termines, haz clic en el botón verde "Commit changes" → "Commit directly to main" → "Commit changes". En 2 minutos el sitio se actualiza en vivo.
Truco: En GitHub, presiona la tecla . (punto) mientras ves tu repo y se abre un editor completo tipo VS Code en el navegador, mucho más cómodo para editar.
2 Cambiar textos básicos

Estos son los campos que más seguido querrás actualizar:

nombre: "Isay", ← el nombre que aparece en el masthead nombre_completo: "Miguel Isay Morales Cortes", headline: "I work with models — the machine learning kind.", tagline: "Data Science · GDL 🇲🇽", bio: "Tu descripción personal aquí.", email: "123miguelisay@gmail.com", instagram: "miguel_isay", ← solo el usuario, sin @ twitter: "miguel_isay", ubicacion: "Guadalajara, Jalisco · México",
Siempre deja las comillas "..." al inicio y al final del texto. Si borras una comilla el sitio puede romperse.
3 Actualizar "En este momento"

Esta sección aparece en la columna derecha del hero. Cámbiala cada vez que tu foco principal cambie.

ahora: [ { icono: "🎓", texto: "Organizando NVIDIA DLI para 250 alumnos" }, { icono: "📊", texto: "Impartiendo taller de Excel Avanzado en CUGDL" }, { icono: "⚙️", texto: "Liderando Odoo ERP en Vibamex" }, { icono: "✈️", texto: "Planeando siguiente viaje tech" } ],

Puedes poner cualquier emoji como ícono. Máximo recomendado: 4 elementos.

4 Agregar un evento nuevo

Copia este bloque completo, pégalo al final del array eventos: (antes del ] de cierre), y llena los campos.

// 1. Busca la sección eventos: [ en el archivo // 2. Ve hasta el último evento (el que cierra con }) // 3. Agrega una coma después del } y pega esto: ,{ tipo: "evento", ← "evento" | "taller" | "logro" destacado: false, ← true = tarjeta grande, false = tarjeta normal icono: "🎯", titulo: "Nombre del evento", fecha: "2026", lugar: "CUGDL, Puerto Vallarta", descripcion: "Describe qué fue y cuál fue tu rol.", roles: ["Organizador"], link: "", ← URL al recurso, o "" si no hay link_texto: "Ver más", stats: [] ← deja [] si no hay estadísticas }
Para un evento con estadísticas (como el de NVIDIA), llena el campo stats así:
stats: [{ num:"50", label:"Alumnos" }, { num:"3", label:"Sesiones" }]
5 Eliminar un evento o proyecto
1
Encuentra el bloque a eliminar
Usa Ctrl+F y busca el título del evento, por ejemplo Expo Osaka.
2
Selecciona todo el bloque
El bloque empieza con ,{ o { y termina con }. Incluye la coma al inicio si la tiene.
3
Borra el bloque y guarda
Selecciona de ,{ hasta el } de cierre y presiona Delete. Commit changes.
Después de borrar, asegúrate de que los bloques que quedan estén separados por coma ,. El último bloque del array NO debe tener coma al final.

7 Subir archivos: PDF, Excel, imágenes

Para que tu sitio tenga archivos descargables (guías, hojas de trabajo, presentaciones):

1
Organiza tus archivos en carpetas
Estructura recomendada:
talleres/excel/guia.pdf
talleres/excel/hoja-trabajo.xlsx
talleres/nvidia/material.pdf
2
Sube los archivos a GitHub
En tu repo → "Add file" → "Upload files" → arrastra tus archivos → escribe la ruta de carpeta si es necesario → "Commit changes".
3
Enlaza el archivo desde el evento o proyecto
En el campo link del evento o proyecto, escribe la ruta relativa:
link: "talleres/excel/guia.pdf"
Estructura de carpetas sugerida para tu taller de Excel:
talleres/ excel/ guia.pdf ← guía del instructor hoja-de-trabajo.xlsx ← ejercicios para alumnos clase-1.pdf ← presentación sesión 1 clase-2.pdf
8 Cambiar tu foto de perfil
1
Prepara la foto
Guárdala como profile.jpg. Formato vertical (portrait) funciona mejor. Tamaño recomendado: mínimo 800×1000px.
2
Súbela a la carpeta img/
En GitHub → "Add file" → "Upload files" → selecciona la foto → en el campo de ruta escribe img/profile.jpg → Commit.
3
Actualiza el campo en el CONFIG
En index.html, el campo ya apunta a img/profile.jpg por defecto. Si quieres usar otro nombre, cámbialo en:
foto: "img/profile.jpg"
